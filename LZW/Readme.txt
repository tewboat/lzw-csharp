Алгоритмы компрессии и декомпрессии работают за O(|data| * log(s)), где data - исходный текст, а s - размер алфавита,
то есть 256. log(s) появляется из-за SortedSet в реализации бора. 

Команды:

1) .\LZW.exe stat [FilePath] 
   Команда выводит информацию о коэффициенте сжатия и времени для указанного файла с разными размерами словаря.
   FilePath  - путь к сжимаемому файлу относительно директории, в которой находится исполняемый файл

   Пример:

    .\LZW.exe stat ..\..\..\data-files\doyle_the_adventures.txt

2) .\LZW.exe compress [FilePath] 
   Команда запускает сжатие файла и записывает сжатый текст в новый файл.
   FilePath  - путь к сжимаемому файлу относительно директории, в которой находится исполняемый файл.
   -o, --output - задает путь к файлу, в который будет записано сжатый текст, по умолчанию файл создается в директории 
   с исходным файлом.
   -d, --dictionarySize - задает максимальный размер словаря, по умолчанию равен 4096

   Пример:
   
    .\LZW.exe compress data-files\doyle_the_adventures.txt -o data-files\doyle_the_adventures.txt.test -d 1000000
   
    '..\..\..\data-files\doyle_the_adventures.txt' 574980 B to 226406 B
    
3) .\LZW.exe decompress [FilePath] [OutputFilePath]
   FilePath  - путь к разжимаемому файлу относительно директории, в которой находится исполняемый файл.
   OutputFilePath - путь к файлу, в который будут записаны расжатые данные.

   Пример:
   
   .\LZW.exe decompress data-files\doyle_the_adventures.txt.test data-files\doyle_the_adventures2.txt
   
   '..\..\..\data-files\doyle_the_adventures.txt.test' successfully decompressed!

Как видно по статистике, чем больше максимальный размер словаря, тем меньше по размеру будет закодированый текст. При
этом при увеличении максимального размера словаря увеличивается время работы алгоритма.
